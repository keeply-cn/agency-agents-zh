# ⚙️ Phase 2 手册 — 基础与脚手架

> **持续时间**：3-5 天 | **智能体**：6 | **关卡守护者**：DevOps 自动化师 + 证据收集者

---

## 目标

构建所有后续工作依赖的技术和运营基础。在添加肌肉之前先让骨架站立。此阶段后，每个开发者都有工作环境、可部署的流水线和设计系统可供构建。

## 前置条件

- [ ] Phase 1 质量关卡通过（架构包已批准）
- [ ] Phase 1 交接包已接收
- [ ] 所有架构文档已定稿

## 智能体激活序列

### 工作流 A：基础设施（第 1-3 天，并行）

#### 🚀 DevOps 自动化师 — CI/CD 流水线 + 基础设施
```
激活 DevOps 自动化师，为 [项目] 设置基础设施。

输入：后端架构师系统架构 + 部署要求
所需交付物：
1. CI/CD 流水线（GitHub Actions / GitLab CI）
   - 安全扫描阶段
   - 自动化测试阶段
   - 构建和容器化阶段
   - 部署阶段（蓝绿或金丝雀）
   - 自动回滚能力
2. 基础设施即代码
   - 环境配置（开发、暂存、生产）
   - 容器编排设置
   - 网络和安全配置
3. 环境配置
   - 密钥管理
   - 环境变量管理
   - 多环境一致性

创建文件：
- .github/workflows/ci-cd.yml（或等效）
- infrastructure/（Terraform/CDK 模板）
- docker-compose.yml
- Dockerfile

格式：工作的 CI/CD 流水线，附 IaC 模板
时间线：3 天
```

#### 🏗️ 基础设施运维师 — 云基础设施 + 监控
```
激活基础设施运维师，为 [项目] 设置监控。

输入：DevOps 自动化师基础设施 + 后端架构师架构
所需交付物：
1. 云资源配置
   - 计算、存储、网络资源
   - 自动扩展配置
   - 负载均衡器设置
2. 监控栈
   - 应用指标（Prometheus/DataDog）
   - 基础设施指标
   - 自定义仪表盘（Grafana）
3. 日志和告警
   - 集中式日志聚合
   - 关键阈值的告警规则
   - on-call 通知设置
4. 安全加固
   - 防火墙规则
   - SSL/TLS 配置
   - 访问控制策略

格式：基础设施就绪报告，附仪表盘访问
时间线：3 天
```

#### ⚙️ 工作室运营 — 流程设置
```
激活工作室运营，为 [项目] 设置流程。

输入：Sprint 排序师计划 + 项目牧羊人协调需求
所需交付物：
1. Git 工作流
   - 分支策略（GitFlow / 主干开发）
   - PR 审查流程
   - 合并策略
2. 沟通渠道
   - 团队渠道设置
   - 通知路由
   - 状态更新频率
3. 文档模板
   - PR 模板
   - Issue 模板
   - 决策日志模板
4. 协作工具
   - 项目板设置
   - 冲刺追踪配置

格式：运营手册
时间线：2 天
```

### 工作流 B：应用基础（第 1-4 天，并行）

#### 🎨 前端开发者 — 项目脚手架 + 组件库
```
激活前端开发者，为 [项目] 进行项目脚手架。

输入：UX 架构师 CSS 设计系统 + 品牌守护者标识
所需交付物：
1. 项目脚手架
   - 框架设置（React/Vue/Angular，按架构）
   - TypeScript 配置
   - 构建工具（Vite/Webpack/Next.js）
   - 测试框架（Jest/Vitest + Testing Library）
2. 设计系统实现
   - 来自 UX 架构师的 CSS 设计令牌
   - 基础组件库（Button、Input、Card、Layout）
   - 主题系统（浅色/深色/系统切换）
   - 响应式工具
3. 应用外壳
   - 路由设置
   - 布局组件（Header、Footer、Sidebar）
   - 错误边界实现
   - 加载状态

创建文件：
- src/（应用源码）
- src/components/（组件库）
- src/styles/（设计令牌）
- src/layouts/（布局组件）

格式：附组件库的工作应用骨架
时间线：3 天
```

#### 🏗️ 后端架构师 — 数据库 + API 基础
```
激活后端架构师，为 [项目] 进行 API 基础。

输入：系统架构规范 + 数据库模式设计
所需交付物：
1. 数据库设置
   - 模式部署（迁移）
   - 索引创建
   - 开发种子数据
   - 连接池配置
2. API 支架
   - 框架设置（Express/FastAPI 等）
   - 匹配架构的路由结构
   - 中间件栈（认证、验证、错误处理、CORS）
   - 健康检查端点
3. 认证系统
   - 认证提供商集成
   - JWT/会话管理
   - 基于角色的访问控制支架
4. 服务通信
   - API 版本设置
   - 请求/响应序列化
   - 错误响应标准化

创建文件：
- api/ 或 server/（后端源码）
- migrations/（数据库迁移）
- docs/api-spec.yaml（OpenAPI 规范）

格式：附数据库和认证的 API 支架
时间线：4 天
```

#### 🏛️ UX 架构师 — CSS 系统实现
```
激活 UX 架构师，为 [项目] 进行 CSS 系统实现。

输入：品牌守护者标识 + 自己的 Phase 1 CSS 设计系统规范
所需交付物：
1. 设计令牌实现
   - CSS 自定义属性（颜色、排版、间距）
   - 附语义命名的品牌色板
   - 附响应式调整的排版比例
2. 布局系统
   - 容器系统（响应式断点）
   - 网格模式（2 列、3 列、侧边栏）
   - Flexbox 工具
3. 主题系统
   - 浅色主题变量
   - 深色主题变量
   - 系统偏好检测
   - 主题切换组件
   - 主题间平滑过渡

创建/更新文件：
- css/design-system.css（或框架中的等效）
- css/layout.css
- css/components.css
- js/theme-manager.js

格式：附主题切换的实现 CSS 设计系统
时间线：2 天
```

## 验证检查点（第 4-5 天）

### 证据收集者验证
```
激活证据收集者，进行 Phase 2 基础验证。

用截图证据验证以下内容：
1. CI/CD 流水线成功执行（显示流水线日志）
2. 应用骨架在浏览器中加载（桌面截图）
3. 应用骨架在移动端加载（移动截图）
4. 主题切换工作（浅色 + 深色截图）
5. API 健康检查响应（curl 输出）
6. 数据库可访问（迁移状态）
7. 监控仪表盘活跃（仪表盘截图）
8. 组件库渲染（组件演示页面）

格式：附截图的证据包
裁决：PASS / FAIL，附具体问题
```

## 关卡决策

**需要双重签署**：DevOps 自动化师（基础设施）+ 证据收集者（视觉）

- **通过**：工作的骨架，附完整 DevOps 流水线 → Phase 3 激活
- **失败**：具体基础设施或应用问题 → 修复并重新验证
