# 🏗️ Phase 1 手册 — 战略与架构

> **持续时间**：5-10 天 | **智能体**：8 | **关卡守护者**：工作室制片人 + 现实检验者

---

## 目标

定义我们要构建什么、如何结构化、成功是什么样子 — 在写第一行代码之前。每个架构决策都有文档记录。每个功能都有优先级。每一美元都有归属。

## 前置条件

- [ ] Phase 0 质量关卡通过（GO 决策）
- [ ] Phase 0 交接包已接收
- [ ] 利益相关者对项目范围达成一致

## 智能体激活序列

### 步骤 1：战略框架（第 1-3 天，并行）

#### 🎬 工作室制片人 — 战略组合对齐
```
激活工作室制片人，对 [项目] 进行战略组合对齐。

输入：Phase 0 高管摘要 + 市场分析报告
所需交付物：
1. 附项目定位的战略组合计划
2. 愿景、目标和 ROI 目标
3. 资源分配策略
4. 风险/回报评估
5. 成功标准和里程碑定义

对齐：组织战略目标
格式：战略组合计划模板
时间线：3 天
```

#### 🎭 品牌守护者 — 品牌标识系统
```
激活品牌守护者，对 [项目] 进行品牌标识开发。

输入：Phase 0 UX 研究（人物角色、旅程地图）
所需交付物：
1. 品牌基础（目标、愿景、使命、价值观、个性）
2. 视觉标识系统（CSS 变量颜色、排版、间距）
3. 品牌声音和信息架构
4. Logo 系统规范（如果是新品牌）
5. 品牌使用指南

格式：品牌标识系统文档
时间线：3 天
```

#### 💰 财务追踪员 — 预算和资源规划
```
激活财务追踪员，对 [项目] 进行财务规划。

输入：工作室制片人战略计划 + Phase 0 技术栈评估
所需交付物：
1. 附类别细分的综合项目预算
2. 资源成本预测（智能体、基础设施、工具）
3. 附盈亏平衡分析的 ROI 模型
4. 现金流时间线
5. 附应急储备的财务风险评估

格式：附 ROI 预测的财务计划
时间线：2 天
```

### 步骤 2：技术架构（第 3-7 天，并行，步骤 1 输出可用后）

#### 🏛️ UX 架构师 — 技术架构 + UX 基础
```
激活 UX 架构师，对 [项目] 进行技术架构。

输入：品牌守护者视觉标识 + Phase 0 UX 研究
所需交付物：
1. CSS 设计系统（变量、令牌、比例）
2. 布局框架（Grid/Flexbox 模式、响应式断点）
3. 组件架构（命名约定、层次结构）
4. 信息架构（页面流程、内容层次）
5. 主题系统（浅色/深色/系统切换）
6. 无障碍基础（WCAG 2.1 AA 基线）

创建文件：
- css/design-system.css
- css/layout.css
- css/components.css
- docs/ux-architecture.md

格式：开发者就绪基础包
时间线：4 天
```

#### 🏗️ 后端架构师 — 系统架构
```
激活后端架构师，对 [项目] 进行系统架构。

输入：Phase 0 技术栈评估 + 合规要求
所需交付物：
1. 系统架构规范
   - 架构模式（微服务/单体/无服务器/混合）
   - 通信模式（REST/GraphQL/gRPC/事件驱动）
   - 数据模式（CQRS/事件溯源/CRUD）
2. 附索引策略的数据库模式设计
3. 附版本控制的 API 设计规范
4. 认证和授权架构
5. 安全架构（纵深防御）
6. 可扩展性计划（水平扩展策略）

格式：系统架构规范
时间线：4 天
```

#### 🤖 AI 工程师 — ML 架构（如适用）
```
激活 AI 工程师，对 [项目] 进行 ML 系统架构。

输入：后端架构师系统架构 + Phase 0 数据审计
所需交付物：
1. ML 系统设计
   - 模型选择和训练策略
   - 数据管线架构
   - 推理策略（实时/批量/边缘）
2. AI 伦理和安全框架
3. 模型监控和重新训练计划
4. 与主应用的集成点
5. ML 基础设施成本预测

条件：仅当项目包含 AI/ML 功能时激活
格式：ML 系统设计文档
时间线：3 天
```

#### 👔 高级项目经理 — 规范到任务转换
```
激活高级项目经理，对 [项目] 创建任务列表。

输入：所有 Phase 0 文档 + 架构规范（可用时）
所需交付物：
1. 综合任务列表
   - 引用规范中的确切需求（无奢侈功能）
   - 每个任务有清晰的验收标准
   - 任务间依赖映射
   - 工作量估算（故事点或小时）
2. 工作分解结构
3. 关键路径识别
4. 实施风险登记册

规则：
- 不要添加规范中没有的功能
- 引用需求的确切文本
- 对工作量估算要现实

格式：附验收标准的任务列表
时间线：3 天
```

### 步骤 3：优先级排序（第 7-10 天，顺序，步骤 2 之后）

#### 🎯 Sprint 排序师 — 功能优先级排序
```
激活 Sprint 排序师，对 [项目] 进行待办优先级排序。

输入：
- 高级项目经理 → 任务列表
- 后端架构师 → 系统架构
- UX 架构师 → UX 架构
- 财务追踪员 → 预算框架
- 工作室制片人 → 战略计划

所需交付物：
1. RICE 评分待办（Reach、Impact、Confidence、Effort）
2. 基于速度估算的冲刺分配
3. 附关键路径的依赖图
4. MoSCoW 分类（Must/Should/Could/Won't）
5. 附里程碑映射的发布计划

验证：工作室制片人确认战略对齐
格式：优先级排序的冲刺计划
时间线：2 天
```

## 质量关卡清单

| # | 标准 | 证据来源 | 状态 |
|---|-----------|----------------|--------|
| 1 | 架构覆盖 100% 的规范需求 | 高级 PM 任务列表与架构交叉引用 | ☐ |
| 2 | 品牌系统完整（logo、颜色、排版、声音） | 品牌守护者交付物 | ☐ |
| 3 | 所有技术组件都有实施路径 | 后端架构师 + UX 架构师规范 | ☐ |
| 4 | 预算已批准且在约束内 | 财务追踪员计划 | ☐ |
| 5 | 冲刺计划基于速度且现实 | Sprint 排序师待办 | ☐ |
| 6 | 安全架构已定义 | 后端架构师安全规范 | ☐ |
| 7 | 合规要求集成到架构中 | 法律要求映射到技术决策 | ☐ |

## 关卡决策

**需要双重签署**：工作室制片人（战略）+ 现实检验者（技术）

- **批准**：携带完整架构包进入 Phase 2
- **修订**：特定项目需要返工（返回相关步骤）
- **重构**：根本性架构问题（重新开始 Phase 1）

## 交接给 Phase 2

```markdown
## Phase 1 → Phase 2 交接包

### 架构包：
1. 战略组合计划（工作室制片人）
2. 品牌标识系统（品牌守护者）

4. CSS 设计系统 + UX 架构（UX 架构师）
5. 系统架构规范（后端架构师）
6. ML 系统设计（AI 工程师 — 如适用）
7. 综合任务列表（高级项目经理）
8. 优先级排序的冲刺计划（Sprint 排序师）

### 给 DevOps 自动化师：
- 来自后端架构师的部署架构
- 来自系统架构的环境要求
- 来自基础设施需求的监控要求

### 给前端开发者：
- 来自 UX 架构师的 CSS 设计系统
- 来自品牌守护者的品牌标识
- 来自 UX 架构师的组件架构
- 来自后端架构师的 API 规范

### 给后端架构师（继续）：
- 数据库模式准备好部署
- API 支架准备好实施
- 认证系统架构已定义
```
