# 🔍 Phase 0 手册 — 情报与发现

> **持续时间**：3-7 天 | **智能体**：6 | **关卡守护者**：高管摘要师

---

## 目标

在投入资源之前验证机会。在理解问题、市场和监管格局之前，不要开始构建。

## 前置条件

- [ ] 项目简报或初始概念已存在
- [ ] 已确定利益相关者发起人
- [ ] 发现阶段预算已批准

## 智能体激活序列

### 第一波：并行启动（第 1 天）

#### 🔍 趋势研究员 — 市场情报负责人
```
激活趋势研究员，对 [项目领域] 进行市场情报收集。

所需交付物：
1. 竞争格局分析（直接 + 间接竞争对手）
2. 市场规模：TAM、SAM、SOM，附方法论
3. 趋势生命周期映射：这个市场处于采用曲线的哪个位置？
4. 3-6 个月趋势预测，附置信区间
5. 该领域的投资和融资趋势

来源：至少 15 个独特的、经过验证的来源
格式：附高管摘要的战略报告
时间线：3 天
```

#### 💬 反馈合成师 — 用户需求分析
```
激活反馈合成师，对 [项目领域] 进行用户需求分析。

所需交付物：
1. 多渠道反馈收集计划（调查、访谈、评论、社交）
2. 现有用户触点的 sentiment 分析
3. 痛点识别和优先级排序（RICE 评分）
4. 功能请求分析及业务价值估算
5. 反馈模式中的流失风险指标

格式：附优先级矩阵的合成反馈报告
时间线：3 天
```

#### 🔍 UX 研究员 — 用户行为分析
```
激活 UX 研究员，对 [项目领域] 进行用户行为分析。

所需交付物：
1. 用户访谈计划（5-10 名目标用户）
2. 人物角色开发（3-5 个主要人物角色）
3. 主要用户流程的旅程地图
4. 竞争对手产品的可用性启发式评估
5. 经统计验证的行为洞察

格式：附人物角色和旅程地图的研究发现报告
时间线：5 天
```

### 第二波：并行启动（第 1 天，独立于第一波）

#### 📊 分析报告员 — 数据格局评估
```
激活分析报告员，对 [项目领域] 进行数据格局评估。

所需交付物：
1. 现有数据源审计（有哪些可用数据？）
2. 信号识别（我们可以衡量什么？）
3. 基线指标建立
4. 数据质量评估，附完整性评分
5. 分析基础设施建议

格式：附信号图的数据审计报告
时间线：2 天
```

#### ⚖️ 法务合规员 — 监管扫描
```
激活法务合规员，对 [项目领域] 进行监管扫描。

所需交付物：
1. 适用的监管框架（GDPR、CCPA、HIPAA 等）
2. 数据处理要求和约束
3. 目标市场的管辖权映射
4. 合规风险评估，附严重程度评级
5. 阻塞性与可管理的合规问题

格式：合规要求矩阵
时间线：3 天
```

#### 🛠️ 工具评估师 — 技术格局
```
激活工具评估师，对 [项目领域] 进行技术格局评估。

所需交付物：
1. 问题领域的技术栈评估
2. 关键组件的自建 vs 购买分析
3. 与现有系统的集成可行性
4. 开源 vs 商业评估
5. 技术风险评估

格式：附建议矩阵的技术栈评估
时间线：2 天
```

## 收敛点（第 5-7 天）

所有六个智能体交付他们的报告。高管摘要师合成：

```
激活高管摘要师以合成 Phase 0 发现。

输入文档：
1. 趋势研究员 → 市场分析报告
2. 反馈合成师 → 合成反馈报告
3. UX 研究员 → 研究发现报告
4. 分析报告员 → 数据审计报告
5. 法务合规员 → 合规要求矩阵
6. 工具评估师 → 技术栈评估

输出：高管摘要（≤500 字，SCQA 格式）
需要决策：GO / NO-GO / PIVOT
包含：量化的市场机会、验证的用户需求、监管路径、技术可行性
```

## 质量关卡清单

| # | 标准 | 证据来源 | 状态 |
|---|-----------|----------------|--------|
| 1 | 市场机会经 TAM > 最低可行阈值验证 | 趋势研究员报告 | ☐ |
| 2 | ≥3 个经验证的用户痛点，附支持数据 | 反馈合成师 + UX 研究员 | ☐ |
| 3 | 未识别出阻塞性合规问题 | 法务合规员矩阵 | ☐ |
| 4 | 关键指标和数据源已识别 | 分析报告员审计 | ☐ |
| 5 | 技术栈可行且已评估 | 工具评估师评估 | ☐ |
| 6 | 高管摘要已交付，附 GO/NO-GO 建议 | 高管摘要师 | ☐ |

## 关卡决策

- **GO**：进入 Phase 1 — 战略与架构
- **NO-GO**：归档发现、记录学习、重定向资源
- **PIVOT**：根据发现修改范围/方向，重新运行针对性发现

```markdown
- [来自法务合规员的监管约束]
- [来自工具评估师的技术约束]

```
