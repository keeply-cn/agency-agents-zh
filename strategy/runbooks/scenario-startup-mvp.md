# 🚀 操作手册：初创公司 MVP 构建

> **模式**：NEXUS-Sprint | **周期**：4-6 周 | **智能体**：18-22 个

---

## 场景

你要构建一个初创公司 MVP——一个需要快速验证产品市场匹配度的新产品。速度很重要，但质量同样重要。你需要在 4-6 周内从想法走向拥有真实用户的上线产品。

## 智能体阵容

### 核心团队（始终活跃）
| 智能体 | 角色 |
|-------|------|
| 智能体编排师 | 流水线控制器 |
| 高级项目经理 | 需求规格转任务 |
| 冲刺优先级师 | 待办事项管理 |
| UX 架构师 | 技术基础 |
| 前端开发者 | UI 实现 |
| 后端架构师 | API 和数据库 |
| DevOps 自动化师 | CI/CD 和部署 |
| 证据收集师 | 每个任务的 QA |
| 现实检验师 | 最终质量关卡 |

### 增长团队（第 3 周 + 激活）
| 智能体 | 角色 |
|-------|------|
| 增长黑客师 | 获客策略 |
| 内容创作师 | 发布内容 |
| 社交媒体策略师 | 社交活动 |

### 支持团队（按需）
| 智能体 | 角色 |
|-------|------|
| 品牌守护者 | 品牌标识 |
| 数据分析报告师 | 指标和仪表盘 |
| 快速原型师 | 快速验证实验 |
| AI 工程师 | 如产品包含 AI 功能 |
| 性能基准师 | 发布前负载测试 |
| 基础设施维护师 | 生产环境设置 |

## 逐周执行

### 第 1 周：发现 + 架构（阶段 0 + 阶段 1 压缩）

```
第 1-2 天：压缩发现
├── 趋势研究师 → 快速竞品扫描（1 天，非完整报告）
├── UX 架构师 → 关键用户流程线框图
└── 高级项目经理 → 需求规格转任务列表

第 3-4 天：架构
├── UX 架构师 → CSS 设计系统 + 组件架构
├── 后端架构师 → 系统架构 + 数据库模式
├── 品牌守护者 → 快速品牌基础（颜色、字体、声音）
└── 冲刺优先级师 → RICE 评分待办事项 + 冲刺计划

第 5 天：基础设置
├── DevOps 自动化师 → CI/CD 流水线 + 环境
├── 前端开发者 → 项目脚手架
├── 后端架构师 → 数据库 + API 脚手架
└── 质量关卡：架构包批准
```

### 第 2-3 周：核心构建（阶段 2 + 阶段 3）

```
冲刺 1（第 2 周）：
├── 智能体编排师管理开发↔QA 循环
├── 前端开发者 → 核心 UI（认证、主视图、导航）
├── 后端架构师 → 核心 API（认证、CRUD、业务逻辑）
├── 证据收集师 → 每个完成任务的 QA
├── AI 工程师 → 如适用则 ML 功能
└── 周末冲刺评审

冲刺 2（第 3 周）：
├── 继续剩余功能的开发↔QA 循环
├── 增长黑客师 → 设计病毒机制 + 推荐系统
├── 内容创作师 → 开始发布内容创作
├── 数据分析报告师 → 设置追踪和仪表盘
└── 周末冲刺评审
```

### 第 4 周：打磨 + 加固（阶段 4）

```
第 1-2 天：质量冲刺
├── 证据收集师 → 完整截图套件
├── 性能基准师 → 负载测试
├── 前端开发者 → 修复 QA 问题
├── 后端架构师 → 修复 API 问题
└── 品牌守护者 → 品牌一致性审计

第 3-4 天：现实检验
├── 现实检验师 → 最终集成测试
├── 基础设施维护师 → 生产就绪
└── DevOps 自动化师 → 生产部署准备

第 5 天：关卡决策
├── 现实检验师裁决
├── 如需要改进：快速修复周期（2-3 天）
├── 如就绪：进入发布
└── 执行摘要生成师 → 利益相关者简报
```

### 第 5-6 周：发布 + 增长（阶段 5）

```
第 5 周：发布
├── DevOps 自动化师 → 生产部署
├── 增长黑客师 → 激活获客渠道
├── 内容创作师 → 发布内容
├── 社交媒体策略师 → 跨平台活动
├── 数据分析报告师 → 实时监控
└── 支持响应师 → 用户支持激活

第 6 周：优化
├── 增长黑客师 → 分析和优化渠道
├── 反馈分析师 → 收集早期用户反馈
├── 实验追踪师 → 启动 A/B 测试
├── 数据分析报告师 → 第 1 周分析
└── 冲刺优先级师 → 计划迭代冲刺
```

## 关键决策

| 决策点 | 时间 | 决策者 |
|---------------|------|-------------|
| 概念通过/不通过 | 第 2 天结束 | 工作室制作人 |
| 架构批准 | 第 4 天结束 | 高级项目经理 |
| MVP 功能范围 | 冲刺规划 | 冲刺优先级师 |
| 生产就绪 | 第 4 周第 5 天 | 现实检验师 |
| 发布时机 | 现实检验师就绪后 | 工作室制作人 |

## 成功标准

| 指标 | 目标 |
|--------|--------|
| 上线产品时间 | ≤ 6 周 |
| 核心功能完成 | MVP 范围的 100% |
| 首批用户入驻 | 发布后 48 小时内 |
| 系统正常运行时间 | 首周 > 99% |
| 收集的用户反馈 | 前 2 周 ≥ 50 条回复 |

## 常见陷阱与缓解措施

| 陷阱 | 缓解措施 |
|---------|-----------|
| 构建过程中的范围蔓延 | 冲刺优先级师执行 MoSCoW——"不会有"就是不会有 |
| 为规模过度工程化 | 快速原型师心态——先验证，后扩展 |
| 为速度跳过 QA | 证据收集师运行于*每个*任务——无例外 |
| 无监控发布 | 基础设施维护师在第 1 周设置监控 |
| 无反馈机制 | 分析 + 反馈收集内置于冲刺 1 |
