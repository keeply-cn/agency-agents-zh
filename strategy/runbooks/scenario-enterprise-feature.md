# 🏢 操作手册：企业级功能开发

> **模式**：NEXUS-Sprint | **周期**：6-12 周 | **智能体**：20-30 个

---

## 场景

你要为现有的企业级产品添加一个重大功能。合规性、安全性和质量关卡是不可协商的。需要多方利益相关者对齐。该功能必须与现有系统无缝集成。

## 执行计划

### 阶段 1：需求与架构（第 1-2 周）

```
第 1 周：利益相关者对齐
├── 项目协调员 → 利益相关者分析 + 沟通计划
├── UX 研究员 → 功能需求的用户研究
├── 法律合规检查师 → 合规要求扫描
├── 高级项目经理 → 需求规格转任务
└── 财务追踪师 → 预算框架

第 2 周：技术架构
├── UX 架构师 → UX 基础 + 组件架构
├── 后端架构师 → 系统架构 + 集成计划
├── UI 设计师 → 组件设计 + 设计系统更新
├── 冲刺优先级师 → RICE 评分待办事项
├── 品牌守护者 → 品牌影响评估
└── 质量关卡：架构评审（项目协调员 + 现实检验师）
```

### 阶段 2：基础建设（第 3 周）

```
├── DevOps 自动化师 → 功能分支流水线 + 功能开关
├── 前端开发者 → 组件脚手架
├── 后端架构师 → API 脚手架 + 数据库迁移
├── 基础设施维护师 → 预发布环境设置
└── 质量关卡：基础验证（证据收集师）
```

### 阶段 3：构建（第 4-9 周）

```
冲刺 1-3（第 4-9 周）：
├── 智能体编排师 → 开发↔QA 循环管理
├── 前端开发者 → UI 实现（逐个任务）
├── 后端架构师 → API 实现（逐个任务）
├── 高级开发者 → 复杂/高级功能
├── 证据收集师 → 每个任务的 QA（截图）
├── API 测试师 → 每个 API 任务的端点验证
├── 实验追踪师 → 关键功能的 A/B 测试设置
│
├── 每两周：
│   ├── 项目协调员 → 利益相关者状态更新
│   ├── 执行摘要生成师 → 执行简报
│   └── 财务追踪师 → 预算追踪
│
└── 附带利益相关者演示的冲刺评审
```

### 阶段 4：加固（第 10-11 周）

```
第 10 周：证据收集
├── 证据收集师 → 完整截图套件
├── API 测试师 → 完整回归套件
├── 性能基准师 → 10 倍流量负载测试
├── 法律合规检查师 → 最终合规审计
├── 测试结果分析员 → 质量指标仪表盘
└── 基础设施维护师 → 生产就绪

第 11 周：最终裁决
├── 现实检验师 → 集成测试（默认：需要改进）
├── 如需修复周期（2-3 天）
├── 重新验证
└── 执行摘要生成师 → 通过/不通过建议
```

### 阶段 5：发布（第 12 周）

```
├── DevOps 自动化师 → 金丝雀部署（5% → 25% → 100%）
├── 基础设施维护师 → 实时监控
├── 数据分析报告师 → 功能采用追踪
├── 支持响应师 → 新功能用户支持
├── 反馈分析师 → 早期反馈收集
└── 执行摘要生成师 → 发布报告
```

## 利益相关者沟通节奏

| 受众 | 频率 | 智能体 | 格式 |
|----------|-----------|-------|--------|
| 执行发起人 | 每两周 | 执行摘要生成师 | SCQA 摘要（≤500 字） |
| 产品团队 | 每周 | 项目协调员 | 状态报告 |
| 工程团队 | 每日 | 智能体编排师 | 流水线状态 |
| 合规团队 | 每月 | 法律合规检查师 | 合规状态 |
| 财务 | 每月 | 财务追踪师 | 预算报告 |

## 质量要求

| 要求 | 阈值 | 验证者 |
|-------------|-----------|-------------|
| 代码覆盖率 | > 80% | 测试结果分析员 |
| API 响应时间 | P95 < 200ms | 性能基准师 |
| 可访问性 | WCAG 2.1 AA | 证据收集师 |
| 安全性 | 零严重漏洞 | 法律合规检查师 |
| 品牌一致性 | 95%+ 遵守 | 品牌守护者 |
| 规格合规 | 100% | 现实检验师 |
| 负载处理 | 10 倍当前流量 | 性能基准师 |

## 风险管理

| 风险 | 概率 | 影响 | 缓解措施 | 负责人 |
|------|------------|--------|-----------|-------|
| 集成复杂性 | 高 | 高 | 早期集成测试，每个冲刺都有 API 测试师 | 后端架构师 |
| 范围蔓延 | 中 | 高 | 冲刺优先级师执行 MoSCoW，项目协调员管理变更 | 冲刺优先级师 |
| 合规问题 | 中 | 关键 | 法律合规检查师从第 1 天开始参与 | 法律合规检查师 |
| 性能回归 | 中 | 高 | 性能基准师每个冲刺测试 | 性能基准师 |
| 利益相关者不对齐 | 低 | 高 | 每两周执行简报，项目协调员协调 | 项目协调员 |
